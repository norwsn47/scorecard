# Backlog
## The Golf Tavern — Bruntsfield Links Scorecard App

Items below are logged for future consideration. None are implemented.

---

### 1. Share / save scorecard

When a game is finished, give players the option to save the completed scorecard as an image or share it directly via the browser's native share sheet on mobile. Should look like a branded Golf Tavern scorecard — not a screenshot of the UI.

---

### 2. Course Map

Players can view the Bruntsfield Links course map before starting a game (Home screen) or mid-round (Scorecard header). In future this should support multiple courses.

---

### 3. Celebratory podium summary

When a game is finished, show a podium-style screen celebrating 1st, 2nd, and 3rd place before or alongside the full scorecard. Should feel like a moment — not just a results table. Golf Tavern branded.

---

### 4. All-time leaderboard

A screen showing top scores and records across all games ever played (lowest round, most wins per player, etc.). Requires database — post-MVP only.

---

### 6. Outbuild credit

A subtle branding line in the footer area of the home screen. Reads: **Scorecard** by Outbuild — "Scorecard" in bold, "by Outbuild" in a lighter muted colour, inline on the same line. Small, centred, not prominent — a credit, not a headline. "Outbuild" should be a link to the Outbuild website URL once that exists.

---

### 5. Analytics

Set up web analytics to track usage. Two parts:

1. Confirm Cloudflare Pages built-in analytics are active for basic traffic data.
2. Evaluate and add a product analytics tool (Plausible, Fathom, or PostHog) to track meaningful in-app events such as games started, games completed, player counts, and scorecard shares.

Must be privacy-friendly — no cookie consent banner required.
